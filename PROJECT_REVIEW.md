# Обзор проекта: Asterisk AI Voice Agent (AVA)

## 1. Общее описание и назначение
Проект **Asterisk AI Voice Agent (AVA)** представляет собой интеллектуального голосового агента, который интегрируется с Asterisk (через интерфейс ARI или AudioSocket) и позволяет обрабатывать входящие и исходящие звонки с помощью ИИ.

Система обеспечивает полный цикл обработки голосового вызова:
- **STT (Speech-to-Text):** Распознавание речи звонящего (Vosk, Deepgram, Google и др.).
- **LLM (Large Language Model):** Генерация ответов на основе контекста разговора (OpenAI, Google Gemini, Ollama, Telnyx).
- **TTS (Text-to-Speech):** Синтез речи для ответа агента (Piper, Silero, ElevenLabs и др.).

## 2. Архитектура системы
Проект построен на базе контейнерной архитектуры (Docker Compose) и состоит из нескольких ключевых компонентов:
- **`ai_engine`**: Ядро системы. Отвечает за оркестрацию вызовов, взаимодействие с Asterisk через ARI, маршрутизацию аудио-потоков (RTP/AudioSocket) и управление пайплайнами обработки речи. Написан на Python с использованием асинхронного фреймворка `asyncio` и веб-сервера `aiohttp`.
- **`local_ai_server`**: Опциональный компонент для локального запуска моделей (LLM, TTS, STT) без привязки к облачным провайдерам, что снижает задержки и повышает конфиденциальность.
- **`admin_ui`**: Веб-интерфейс администратора для настройки конфигураций, провайдеров и просмотра истории вызовов.

## 3. Анализ кодовой базы (src/)
- **Движок (`engine.py`, `rtp_server.py`, `ari_client.py`)**:
  - Реализован мощный клиент для работы с Asterisk REST Interface (`ari_client.py`).
  - Поддержка RTP сервера (`rtp_server.py`) для обмена аудио.
  - Движок (`engine.py`) управляет жизненным циклом вызовов, поддерживает интеграцию с WebRTC VAD для точного определения голосовой активности.
- **Пайплайны и провайдеры (`pipelines/`, `providers/`)**:
  - Гибкая архитектура адаптеров (`PipelineOrchestrator`). Поддерживаются как классические пошаговые пайплайны (STT -> LLM -> TTS), так и нативные голосовые модели (например, OpenAI Realtime API, Google Live API).
- **Интеграции и инструменты (`tools/`, `mcp/`)**:
  - Агент может использовать инструменты во время звонка (In-Call Tools): переводить звонки, проверять статусы экстеншенов, завершать вызов.
  - Поддержка пре- и пост-колл вебхуков (Pre-Call / Post-Call Hooks) для интеграции с внешними CRM и системами аналитики.
  - Интеграция с MCP (Model Context Protocol).

## 4. Качество кода и тестирование
- В проекте используется `pytest` для юнит- и интеграционного тестирования (`tests/`).
- Покрытие тестами (около 27-29%) настроено в CI/CD (GitHub Actions) с жестким порогом отсечения (26%).
- Код хорошо структурирован, активно используется типизация (Pydantic, typing), логирование (`structlog`) и метрики (Prometheus).

## 5. Документация и стандарты
- Проект отлично задокументирован. Имеется обширная директория `docs/` с инструкциями по деплою (в т.ч. интеграция с FreePBX и Vicidial), настройке провайдеров, и архитектурными деталями.
- Присутствуют специфические инструкции (`CLAUDE.md`, `AGENTS.md`), описывающие стандарты оформления кода, в том числе по ГОСТ (ЕСПД), что указывает на серьезный подход к разработке и сдаче проекта заказчикам.

## 6. Сильные стороны и потенциал
- **Модульность:** Легко добавлять новых провайдеров ИИ и новые инструменты.
- **Гибридность:** Возможность работать как с облачными (OpenAI, Google), так и с полностью локальными (Ollama, Vosk, Piper) решениями.
- **Производительность:** Переход на AudioSocket и оптимизация ресемплинга аудио (NumPy) снижают задержки и улучшают качество звука.
- **Готовность к Production:** Наличие CLI-инструментов (`agent cli`), UI для администрирования, метрик Prometheus и скриптов автоматической проверки (`preflight.sh`).

## 7. Рекомендации по развитию
- **Увеличение покрытия тестами:** Текущая цель в 40% покрытия является приоритетной, особенно для ядра движка (`engine.py`).
- **Расширение возможностей Admin UI:** Добавление визуального конструктора диалогов или более детальной аналитики по каждому вызову прямо из интерфейса.
- **Оптимизация задержек:** Продолжение работы над оптимизацией VAD (barge-in) и стримингового TTS для достижения минимально возможных задержек (sub-500ms).

---
**Итог:** Asterisk AI Voice Agent — это зрелый, активно развивающийся Open-Source проект с продуманной архитектурой. Он готов к интеграции в реальные бизнес-процессы (телемаркетинг, техподдержка) и имеет отличный потенциал благодаря своей гибкости и независимости от вендоров (local-first подход).
