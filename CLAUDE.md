# callagent — инструкции проекта

## Оформление по ГОСТ (ЕСПД)

Все технические артефакты в проекте оформляются по стандартам ЕСПД (ГОСТ 19.\*).
Это требование заказчика (MMVS, работа по 44-ФЗ).

### Документация на модули (ГОСТ 19.402 «Описание программы»)
Структура: «Общие сведения» → «Функциональное назначение» → «Описание
логической структуры» → «Используемые технические средства» → «Вызов
и загрузка» → «Входные данные» → «Выходные данные».

### Тест-отчёты (ГОСТ 19.301 «Программа и методика испытаний»)
Структура: «Объект испытаний» → «Цель испытаний» → «Требования» →
«Состав и порядок» → «Методы» → «Результаты».

### Руководство оператора (ГОСТ 19.505)
Структура: «Назначение программы» → «Условия выполнения» → «Выполнение»
→ «Сообщения оператору».

### Правила оформления
- Имена документов: `desc_<module>.md`, `test_report_<module>.md`,
  `manual_<module>.md`.
- Даты: ГГГГ-ММ-ДД (ISO 8601 / ГОСТ ИСО 8601-2001).
- Версии: семантические (MAJOR.MINOR.PATCH) в шапке документа.
- Документация — на русском языке.
- Идентификаторы в коде — английские (Python/JS норма), комментарии —
  на русском только когда поясняют «почему» (скрытое ограничение,
  инвариант, обход бага).
- Логи и UI-сообщения — на русском, формат: «Уровень: краткое описание
  [деталь]». Уровни: ОТЛАДКА / ИНФО / ПРЕДУПРЕЖДЕНИЕ / ОШИБКА / КРИТ.

### Конфликт с практикой 2026
Если ГОСТ 19 требует артефакты, которые сегодня не делают (рукописные
блок-схемы, листинги через P-схемы) — отдавать приоритет духу стандарта
(полнота, формальные разделы), а не букве.

## Архитектура (актуальное состояние)

См. memory: `MEMORY.md`, `project_ollama_pipeline.md`,
`project_direct_audiosocket.md`, `project_voice_pipeline_optimization.md`.

- Docker Compose: `ai_engine`, `local_ai_server`, `admin_ui`.
- Admin UI: http://localhost:3003 (admin/admin).
- LLM: Ollama (qwen2.5:7b, fast_local) или local_ai_server (Mistral 7B GGUF, GPU).
- TTS: Piper, XTTS (клон голоса Андреева, в работе).
- STT: Vosk-big / Silero VAD.
- Транспорт: app_audiosocket (без bridge), SLIN.

## Команды

- Сборка: `docker compose build`
- Запуск: `docker compose up -d`
- Логи: `docker compose logs -f <service>`
- Тесты: `pytest tests/ -v`
- Локальный AI-сервер вручную: см. `MEMORY.md` → «local_ai_server Container Run Command».
