## 1. Общие сведения

**Обозначение и наименование программы.** AI Telemarketer (далее — Программа) — модуль голосового агента телемаркетинга в составе изделия callagent.

**Программное обеспечение, необходимое для функционирования.**
- Операционная система Linux (ядро ≥ 6.0).
- Python 3.10 или выше.
- Сервер инференса больших языковых моделей Ollama (≥ 0.1.40).
- Графический ускоритель NVIDIA с поддержкой CUDA 12.4 и не менее 8 ГБ видеопамяти.
- Пакеты: `faster-whisper`, `TTS` (Coqui), `fastapi`, `uvicorn`, `aiohttp`, `numpy`, `scipy`, `soundfile`, `pydantic`. Полный перечень — в `ai_telemarketer/requirements.txt`.

**Языки программирования.** Python (исходный код Программы), JavaScript (клиентская страница `voice_lab.html`).

## 2. Функциональное назначение

**Назначение.** Программа реализует автоматизированный исходящий телефонный обзвон с возможностью двустороннего голосового диалога между клиентом и виртуальным телемаркетологом. Управление диалогом построено на конечном автомате из пяти состояний: INTRO, QUALIFY, VALUE, OBJECTION, CLOSE.

**Сведения о функциональных ограничениях применения.**
- Поддерживается русский язык диалога. Голос синтезируется на основе fine-tuned копии голоса Андреева (модель XTTS v2).
- Один экземпляр Программы обслуживает не более одного диалога одновременно (модель XTTS не потокобезопасна; параллельный синтез сериализован внутренним мьютексом).
- При отсутствии доступа к сервису Ollama Программа не запускается (см. healthcheck в разделе 5).

## 3. Описание логической структуры

**Алгоритм работы.** Программа работает в двух режимах: HTTP API и WebSocket.

В режиме HTTP API:
1. Получает аудио и текущее состояние диалога.
2. Распознаёт речь (STT, Faster-Whisper).
3. Передаёт распознанный текст и состояние в LLM (Ollama, модель qwen2.5:1.5b или qwen2.5:7b).
4. LLM возвращает ответ и следующее состояние в формате JSON.
5. Синтезирует речь по ответу (TTS, XTTS v2 + reference WAV Андреева).
6. Возвращает аудиоданные и метаданные в HTTP-ответе.

В режиме WebSocket:
1. Принимает поток сырых PCM16-фреймов 16 кГц.
2. По энергии сигнала определяет конец фразы (silence_threshold = 1000, silence_duration > 1.2 с при минимальном объёме буфера 16000 байт).
3. Запускает шаги STT → LLM → TTS.
4. Передаёт TTS-аудио клиенту по предложениям, что снижает задержку до первого фрейма (TTFB).

**Используемые методы.**
- **STT.** Faster-Whisper (модель `small` или `medium`), beam_size=5, режим распознавания русского языка.
- **LLM.** Конечный автомат на промпт-инжиниринге. Системный промпт + инструкция по текущему состоянию + реплика пользователя. Формат ответа `format=json` (Ollama-side validation).
- **TTS.** Coqui XTTS v2 с дообученным чекпойнтом. Reference WAV-файл задаёт тембр голоса. Выход — float32 массив 24 кГц, кодируется в WAV PCM16 для HTTP-отдачи или в чанковый поток для WebSocket.

**Связь с другими программами.**
- Отдельный сервис, не интегрирован в основной пайплайн `src/`.
- Использует Ollama по сети (по умолчанию `http://localhost:11434`).
- Опциональная связка с Asterisk — модуль `telephony/` (в текущей версии не подключён к `main.py`; запланировано в версии 0.3).

## 4. Используемые технические средства

| Компонент | Минимум | Рекомендуется |
|-----------|---------|---------------|
| Процессор | Intel/AMD x86-64, 4 ядра | Intel Core Ultra 7 или эквивалент, 8 ядер |
| ОЗУ | 16 ГБ | 32 ГБ |
| Видеоадаптер | NVIDIA с 6 ГБ VRAM, Compute Capability ≥ 7.0 | NVIDIA RTX 5060 (8 ГБ, sm_120) — эталонная конфигурация |

## 5. Вызов и загрузка

**Способ вызова.**
```bash
cd /home/dmitriy/work/callagent/ai_telemarketer
source ../venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8000
```

**Параметры окружения.**
| Переменная | По умолчанию | Назначение |
|------------|--------------|------------|
| `TELEMARKETER_OLLAMA_URL` | `http://localhost:11434/api/generate` | URL Ollama-инференса |
| `TELEMARKETER_OLLAMA_MODEL` | `qwen2.5:1.5b` | Имя модели в Ollama |
| `TELEMARKETER_OLLAMA_TIMEOUT_SEC` | `30` | Таймаут запроса в секундах |
| `TELEMARKETER_XTTS_MODEL_DIR` | `/home/.../models/xtts_v2` | Каталог с config.json и весами XTTS |
| `TELEMARKETER_XTTS_SPEAKER_WAV` | `/home/.../models/andreev_voice.wav` | Reference WAV-файл |
| `TELEMARKETER_STT_MODEL_SIZE` | `small` | Размер модели Faster-Whisper |
| `TELEMARKETER_STT_DEVICE` | `cuda` | Устройство для STT (`cuda` / `cpu`) |

**Этапы загрузки.**
1. Healthcheck Ollama (`GET /api/tags`, проверка наличия модели). При неудаче — старт прерывается.
2. Проверка существования каталога модели XTTS и reference WAV.
3. Загрузка STT (≈10 с, ~1 ГБ VRAM).
4. Загрузка XTTS (≈15 с, ~5 ГБ VRAM) и однократный прогрев фразой «Привет.».
5. Готовность к запросам.

## 6. Входные данные

| Эндпоинт | Метод | Тело | Назначение |
|----------|-------|------|-----------|
| `/stt` | POST | `multipart/form-data` (файл WAV/MP3) | Распознавание речи |
| `/generate_response` | POST | JSON `{user_text, current_state}` | Только LLM |
| `/tts` | POST | JSON `{text}` | Только синтез |
| `/dialog_step` | POST | `multipart/form-data` (audio + current_state) | Полный цикл STT→LLM→TTS |
| `/ws` | WebSocket | бинарный PCM16 16 кГц | Потоковый диалог |
| `/health` | GET | — | Готовность сервиса |

## 7. Выходные данные

| Эндпоинт | Формат ответа |
|----------|---------------|
| `/stt` | `application/json`, поле `text` |
| `/generate_response` | `application/json`, поля `response`, `next_state` |
| `/tts` | `audio/wav`, файл XTTS WAV PCM16 24 кГц |
| `/dialog_step` | `audio/wav` + заголовки `X-User-Text`, `X-Agent-Text`, `X-Next-State` |
| `/ws` | JSON-сообщения `debug`, `stt_result`, `llm_response`, `error` + бинарные WAV-фреймы |
| `/health` | HTTP 200 / 503, JSON `{status, stt, llm, tts}` |
